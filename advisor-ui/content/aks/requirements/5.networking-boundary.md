---
title: Network Security Perimeters
spec:
  question: What are your network boundary requirements?
  title: Network Security Perimeters
  description: |
    Determines whether cluster infrastructure uses public or private networking. Remember that public does not mean insecure — your API server and services are still secured via credentials and RBAC.
  question_type: radio
  answers:
  - key: public
    label: Public (Both Planes)
    description: |
      Both the control plane (API server) and data plane (workload traffic) are accessible over public networks. Simplest to set up and manage.
    implications:
      - control-plane: public
      - data-plane: public
    waf_baseline:
      security: 0
      operations: -5
      cost: 0

  - key: private-data-plane
    label: Private Data Plane
    description: |
      Workload traffic runs within your private VNet. The API server and Microsoft-managed components remain publicly accessible. Good balance of security and operational simplicity.
    implications:
      - control-plane: public
      - data-plane: private
    waf_baseline:
      security: 5
      operations: 3
      cost: 2

  - key: fully-private
    label: Fully Private (Both Planes)
    description: |
      Both control plane and data plane are private. The API server is only reachable inside your VNet. Non-cluster operations traffic routes through a firewall. Requires VPN or jumpbox for cluster access.
    implications:
      - control-plane: private
      - data-plane: private
    waf_baseline:
      security: 10
      operations: 8
      cost: 5
---
