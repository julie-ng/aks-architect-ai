title: Identity & Access
designer:
  title: Identity & Access Configuration
  description: |
    Determines how users, cluster components, and workloads authenticate and authorize against Azure and Kubernetes resources.
  question: |
    Which identity and access features do you need?
  question_type: checkbox
  reference:
    title: Access and identity options for AKS
    url: https://learn.microsoft.com/en-us/azure/aks/concepts-identity
  answers:
  - key: entra_integration
    label: Microsoft Entra Integration
    description: |
      Integrate cluster authentication with Microsoft Entra ID (formerly Azure AD). Users and groups authenticate using their corporate identity.
    waf_impact:
      security: 7
      operations: 1

  - key: azure_rbac
    label: Azure RBAC for Kubernetes Authorization
    description: |
      Use Azure role assignments to control access to Kubernetes resources. Centralizes authorization in Azure instead of managing Kubernetes RBAC directly.
    waf_impact:
      security: 7
      operations: 0 # use built-in roles

  - key: managed_identities
    label: Managed Identities for AKS Components
    description: |
      Use managed identities instead of service principals for cluster components. System-assigned identities are simpler but leave dangling role assignments on deletion. User-assigned identities give you control over the identity lifecycle and can be shared across resources.
    waf_impact:
      security: 8
      operations: -2

  - key: workload_identity
    label: Workload Identity
    description: |
      Allows pods to authenticate to Azure services using federated identity credentials. Eliminates the need for secrets or connection strings in workloads.
    waf_impact:
      security: 10
      operations: 2
