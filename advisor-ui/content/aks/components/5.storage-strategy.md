---
title: Storage Strategy
designer:
  title: Workload Storage Strategy
  description: |
    How your workloads handle data affects networking, security, and identity configuration. Managed databases require private endpoints and workload identity. Stateful workloads require persistent volume configuration and backup planning.
  question: |
    What is the storage strategy for your workloads?
  question_type: radio
  reference:
    title: Storage options for applications in AKS
    url: https://learn.microsoft.com/en-us/azure/aks/concepts-storage
  answers:
  - key: stateless
    label: Stateless
    description: |
      Workloads do not persist data. No storage or external database configuration required.
    waf_impact:
      reliability: 0
      security: 0
      cost: 0
      operations: 0

  - key: managed_services
    label: Managed Azure Services (PaaS)
    description: |
      Workloads connect to managed databases and storage services (e.g. Azure SQL, Cosmos DB, Azure Storage). Requires secure configuration.
    waf_impact:
      reliability: 5
      security: 5
      cost: 0 # scoped to AKS
      operations: -5

  - key: stateful
    label: Stateful with Persistent Volumes
    description: |
      Workloads mount persistent storage on the cluster using Azure Disks or Azure Storage (Files and Blobs). Requires storage class configuration, backup strategy, and capacity planning.
    waf_impact:
      reliability: -3
      security: -3
      cost: 3
      operations: 8
